# Принятые решения и осознанные риски

Журнал долговечных решений (ADR-лайт). Сюда попадает то, что нельзя
вывести из кода: «почему так», принятые компромиссы и остаточные риски.
Полные тексты аудита 2026-06-05 и ревью 2026-06-01 удалены из репозитория
(есть в git-истории: `docs/AUDIT_2026-06.md`, `docs/CODE_REVIEW.md`);
всё существенное из них — здесь.

## Архитектурные и продуктовые решения

- **Все `User` — полностью доверенные администраторы.** Ролевой модели нет
  намеренно; создание учётки = выдача полного доступа (включая Horizon и
  выпуск токенов). Escape hatch при необходимости — флаг `users.is_admin`.
  Детали: [04-admin.md](./04-admin.md).
- **Soft-deleted ссылка = отключена**: публичный резолв даёт 404, админ
  видит trashed. Клики и колбеки — исторические факты, **переживают
  ссылку** (в джобах `withTrashed()`, FK `restrictOnDelete`).
- **Связи моделей — только трейтами** в `app/Models/Relations/`, даже
  однострочные (осознанно вопреки рекомендации аудита «заинлайнить»).
- **Клик засчитывается на рендере интерстишала** (delayed/manual) и для
  ботов/prefetch — продуктовое решение; боты помечаются флагом
  (этап 1 [планов](./07-plans.md)), а не выбрасываются.
- **Футер логина рендерится как HTML без экранирования** — принято by
  design: `content.footer` задаёт только доверенный админ. Пересмотреть,
  если появятся недоверенные операторы.
- **Генерация кода в `Link::booted()`** — принято как идиоматичное
  (аудит предлагал перенос в `LinkCreator`; отклонено).
- **IP-pinning в колбеках (TOCTOU / DNS-rebinding) не реализован
  намеренно**: `callback_url` задаёт только админ, follow-redirect
  отключён, а чистая реализация (CURLOPT_RESOLVE) нетестируема через
  `Http::fake`. Включить при переходе на пользовательские `callback_url`.
- **trustProxies = приватные подсети** (`10/8`, `172.16/12`, `192.168/16`),
  не `*`. В проде желательно сузить до реального прокси. Остаточный риск:
  `172.16/12` покрывает все docker-bridge-сети.
- **`HASHIDS_SALT` независим от `APP_KEY`** — ротация ключа приложения не
  меняет коды ссылок. `links.code` — `varchar(16)`: потолок Hashids снят
  на весь диапазон bigint (9-символьный код начинается с id ~52,5 млрд).
- **Корневой маршрут `GET /{code}`** ограничен constraint'ом по
  алфавиту/длине кода; проверено, что Hashids не кодирует id в
  зарезервированные слаги Filament (`login`, `services`, …). Остаточный
  риск затенения — только при смене алфавита.
- **`round_robin` доменов — намеренно без курсора**: порядок выводится из
  истории ссылок (`MAX(links.created_at)`), поэтому самокорректируется при
  добавлении/удалении доменов. Под нагрузкой два одновременных запроса
  могут получить один домен — принятый best-effort компромисс; строгая
  гарантия потребовала бы таблицу-курсор с `lockForUpdate`. Статистика
  ротации глобальная (по всем сервисам): домен — общий ресурс.
- **Проверено на рантайме** (не менять «на всякий случай»):
  `firstOrCreate` в Laravel 13 race-safe через `createOrFirst` + savepoint;
  Filament v5 `unique()` игнорирует текущую запись на edit по умолчанию.

## Продуктовые решения 2026-07-11 (разбор каталога competitor)

Обоснования — [10-competitor-catalog-review.md](./10-competitor-catalog-review.md).

- **Колбеки по бот-кликам отправляются всегда, с меткой `is_bot` в корне
  payload** (клиентский ключ `is_bot` перекрывается). Партнёр решает сам;
  ложноположительная детекция не теряет конверсию.
- **A/B sticky — детерминированный хэш `(ip, user_agent, link_id)`**
  против весов; без cookie и состояния. Компромисс: смена IP/браузера
  может дать другой вариант.
- **IP хранится сырым + плановая чистка** (ориентир 90 дней): сырой IP
  нужен антифроду и колбекам партнёров; бессрочное хранение — нет.
- **Кастомные слаги не делаем**: ссылки создаются программно, инвариант
  «код из Hashids, коллизий нет» сохраняется. Вернуться при явном запросе
  клиента.
