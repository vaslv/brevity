# Brevity — документация

Сервис коротких ссылок с маршрутизацией по правилам, учётом кликов и
исходящими колбеками. Laravel 13 + Filament 5 + PostgreSQL, запускается
под Laravel Sail.

Эта папка — канонический справочник для людей и AI-агентов. Оба должны
уметь ответить на «как работает X?» не грепая кодбазу, и понимать, где
вносить изменения.

## Карта (в порядке чтения)

1. [01-architecture.md](./01-architecture.md) — стек, потоки запросов,
   подсистемы, модель данных, раскладка каталогов.
2. [02-glossary.md](./02-glossary.md) — двуязычная (EN ↔ RU)
   терминология. **Начинай отсюда, если не уверен, как что-то назвать.**
3. [03-api.md](./03-api.md) — полный справочник по HTTP API: токены,
   создание ссылок, домены/группы, колбеки, payload'ы для авторов SDK.
4. [04-admin.md](./04-admin.md) — админка Filament: ресурсы, навигация,
   соглашения по i18n.
5. [05-development.md](./05-development.md) — запуск, тесты, стиль,
   релизы, git-соглашения.
6. [06-deploy.md](./06-deploy.md) — прод-топология, CI/CD, ручные
   операции на сервере.
7. [07-plans.md](./07-plans.md) — все активные планы развития (роадмап
   этапов + детальные планы).
8. [08-decisions.md](./08-decisions.md) — принятые решения, осознанные
   риски, реестр находок прошлых аудитов (на него ссылаются
   регресс-тесты).
9. [09-competitor-catalog.md](./09-competitor-catalog.md) — каталог
   функциональности competitor: справочное сырьё текущей задачи развития
   (ID-возможности, на которые ссылаются разбор и планы).
10. [10-competitor-catalog-review.md](./10-competitor-catalog-review.md) —
    критический разбор каталога: тиры ценности для Brevity, отклонённое
    с обоснованиями.

## Поддержание в актуальном виде

Эти файлы — часть проекта. Обновляй их в том же коммите, где меняется
поведение или терминология; документация, разошедшаяся с реальностью, —
баг. Новые решения и компромиссы — в 08-decisions.md. Выполненный или
устаревший документ удаляется (git-история его сохранит), предварительно
отдав долговечные решения в 08-decisions.md.
