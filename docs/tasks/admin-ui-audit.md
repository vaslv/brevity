# Аудит админки — план работ

Источник: внешний отчёт по UX-аудиту админки (апрель 2026).
Ниже — фильтрация пунктов отчёта: что берём, что пропускаем и почему.
Решаем только штатными средствами Filament 5, без заказной UI-логики
и без ломки доменной модели.

## Принципы отбора

- **Берём** — реальный UX-пробел, фиксится стандартным Filament API
  (`->helperText()`, виджеты, `IconColumn`/`TextColumn`, фильтры,
  `navigationBadge`, `Action::requiresConfirmation()` и т.п.).
- **Пропускаем** — поведение по умолчанию Filament, косметика, пункты
  требующие доменных изменений или сторонних парсеров, пункты,
  противоречащие уже зафиксированной терминологии в
  [GLOSSARY.md](./../GLOSSARY.md).

## Берём в работу

### 1. Dashboard: виджеты и зачистка

Заменить стоковый Dashboard на содержательный.

- Убрать виджет `FilamentInfoWidget` (версия Filament + ссылки на
  GitHub). Инфа о стеке — не для оператора.
- Добавить `StatsOverviewWidget` с карточками: всего ссылок, кликов
  сегодня, кликов за неделю, активных сервисов.
- Добавить `ChartWidget` «Клики по дням» (последние 14/30 дней).
- Реализация: `app/Filament/Widgets/`, зарегистрировать в
  `MainPanelProvider::panel()` через `->widgets([...])`,
  `->discoverWidgets()` при желании.

### 2. Helper text на ключевых полях форм

Добавить `->helperText(__('...'))` на нетривиальных полях форм и
`->hint()` где уместно. Строки — в соответствующих lang-файлах под
`fields.<key>_help`.

Поля-кандидаты:
- `Service.callback_url` — что это, когда дёргается.
- `Link.callback_data` — JSON-шаблон с `{{click.*}}` / `{{link.*}}`.
- `Link.forward_query` — «передавать query-параметры в целевой URL».
- `Rule.transition_mode` — короткое описание режимов.
- `Rule.priority` — что делает порядок.
- `Condition.type` / `data.before` — смысл поля.

### 3. Таблица Links: показать главное

- Новая колонка «Short URL» — `TextColumn` с computed-значением
  `{domain}/{code}`, `copyable()` для копирования в один клик.
- Новая колонка «Клики» — `TextColumn` на базе `withCount('clicks')`,
  сортируемая.
- `forward_query`: заменить текущую `IconColumn` на `IconColumn` с
  явными true/false-иконками (check/xmark) + `tooltip` «Передавать
  query-параметры». Альтернатива — `TextColumn` «Да/Нет».
- `Title`: скрыть из дефолтного набора колонок, если почти всегда
  пустое (оставить toggleable + `->toggledHiddenByDefault()`).

### 4. Таблица Links: заголовочное действие «Копировать короткую ссылку»

На `ListLinks` / в row actions — `Action` с иконкой clipboard,
кладёт `{domain}/{code}` в буфер (через JS hook или copyable column
— см. п.3).

### 5. ViewLink / ViewService: отображать готовый короткий URL

В `LinkInfolist` добавить `TextEntry` «Короткая ссылка» со значением
`{domain}/{code}`, `copyable()`, плюс действие-суффикс «Открыть»
(`Action::url(..., shouldOpenInNewTab: true)`).

ViewService: пустой белый блок — проверить инфолист на мусорные пустые
секции / `Section` без `schema`, убрать.

### 6. Delete action на EditLink: подтверждение

Уже есть `DeleteAction` в header — добавить
`->requiresConfirmation()->modalHeading(...)` с явной формулировкой,
указывающей код ссылки. Так оставляем стандартное место кнопки
(Filament-конвенция), но делаем промах дороже.

### 7. Clicks: фильтры и график

- Фильтры: по `link_id` (`SelectFilter` с search), по `service_id`,
  по датам (`Filter` с `DatePicker`).
- Починить колонку Link: сейчас пустая. Скорее всего в
  `ClicksTable` колонка не ссылается на существующий relation — см.
  `app/Filament/Resources/Clicks/Tables/ClicksTable.php`.
- Виджет-график «Клики по дням» для страницы `ListClicks` (header
  widget).
- `user_agent`: колонка `limit(60)->tooltip(fn ($state) => $state)`,
  чтобы полный UA был в tooltip'е. Парсить UA в браузер/ОС —
  отдельная задача (нужен парсер), сюда не берём.

### 8. Sidebar: бейджи-счётчики

На ресурсах с полезными счётчиками — `public static function getNavigationBadge(): ?string`:
- Links — общее количество.
- Clicks — за сегодня.
- Callbacks — в статусе `failed` (с `getNavigationBadgeColor() = 'danger'`).

### 9. Группировка верхнего уровня

Services, Links, Domains сейчас без группы. Добавить группу
«Основное» (`navigation.groups.main`) и проставить
`getNavigationGroup()` этим трём ресурсам. Перевод в
`lang/{en,ru}/navigation.php`. Глоссарий: добавить `main` в таблицу
«Группы навигации».

### 10. Фильтр-индикатор «0»

Таблицы, где `filters()` непустой, показывают счётчик активных
фильтров. Проверить: если Filament в нашей версии рисует «0» при
отсутствии активных — посмотреть `->hiddenLabel()` / настройки
`filtersTriggerAction()`. Мелочь, но трогаем заодно.

## Пропускаем

| Пункт отчёта | Почему не берём |
|---|---|
| Кнопка «Удалить» рядом с «Просмотр» опасна | Штатное место в Filament. Решаем через `requiresConfirmation` (п.6), не перенос. |
| Поле «Целевой URL» прямо в форме Link | Конфликтует с доменной моделью: URL живёт в `Rule`, у одной ссылки может быть несколько правил. Нельзя сделать «штатно», это перепроектирование UX и бизнес-логики. Отдельная задача, не сюда. |
| Показ `Code` при создании | `Code` генерится в `LinkCreator`, до сохранения его нет. Заменяется п.5 (короткий URL виден сразу на ViewLink после создания). |
| «Создать и Создать ещё» — название | Строка из штатных переводов Filament `vendor/filament/*/lang/ru/`. Не переопределяем ради косметики. |
| Переименовать «Справочники» в «Правила» | Конфликт с глоссарием: `Rule` — уже отдельная сущность. Отбой. |
| Логотип не ведёт на Dashboard | В Filament логотип — ссылка на home URL панели по умолчанию. Либо уже работает, либо ломается из-за кастомного view — проверяем мимоходом, отдельного пункта не делаем. |
| Breadcrumb «Список» | Штатная строка Filament. Косметика. |
| Пагинация при 1 записи | Штатное поведение. Косметика. |
| Placeholder глобального поиска | Глобальный поиск сейчас вообще не настроен на ресурсах (`getGloballySearchableAttributes`). Это отдельная задача — либо включаем поиск по Link/Service осмысленно, либо не трогаем. Сюда не берём. |
| Tooltip иконки настройки колонок | Штатное поведение Filament. |
| IP/User agent «без пояснения назначения» | Это справочники-дедуп для аналитики, не чёрные/белые списки. Решается `helperText` / `Section::description` на индексе — мелочь, можно вместе с п.2 если дойдут руки, отдельный пункт не заводим. |
| Парсинг User Agent (Chrome 146 / macOS) | Нужен сторонний парсер (`jenssegers/agent` или аналог) и ADR по хранению. Не «из коробки Filament». Отдельная задача. |
| URLs в справочнике — связь с Link/Service | Technically связь many-to-many через `rules` таблицу, осмысленно только как drill-down. Можно добавить колонку «Используется в N ссылках» (`withCount`) — берём как вторичный пункт в рамках п.3, не отдельным пунктом. |

## Порядок работ

Предлагаемая нарезка коммитов, каждый самодостаточен и ревёртабелен:

1. Dashboard: убрать FilamentInfoWidget, добавить stats + chart. (п.1)
2. Helper text на формах. (п.2)
3. Таблица Links: short URL, clicks count, forward_query fix. (п.3, п.4)
4. ViewLink / ViewService: короткий URL + пустой блок. (п.5)
5. EditLink: подтверждение удаления. (п.6)
6. Clicks: фильтры, фикс Link-колонки, header chart, UA tooltip. (п.7)
7. Sidebar: navigation badges. (п.8)
8. Группа «Основное» + перенос Services/Links/Domains + обновление
   глоссария. (п.9)
9. Фильтр-индикатор — заметка, трогаем если видна проблема. (п.10)

Каждый коммит — с соответствующими правками `lang/en` + `lang/ru` в
том же коммите (см. `DEVELOPMENT.md`).

## Что нужно от пользователя

- Подтверждение порядка и того, что ничего важного не пропущено.
- Решение по двум пунктам из «Пропускаем», если хочется их вернуть:
  - Парсинг User Agent (нужен пакет).
  - Настройка глобального поиска по ресурсам.
